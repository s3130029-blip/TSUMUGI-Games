# 要件定義書 — タマゴガチャ（仮）

> ブラウザで動く、子ども向けタマゴガチャ（カプセルトイ）ゲーム。
> タマゴをタップ → ガタガタ → パカッ → モノが生まれる、を中心とした演出重視の作品。

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| 名称 | タマゴガチャ（正式名称は未定。仮称） |
| ジャンル | タップ式ガチャ + コレクション（図鑑） |
| 対象ユーザー | 未就学〜低学年の子ども（保護者が横で見ている想定） |
| 主な目的 | 「タップする楽しさ」「何が出るかのワクワク」「集める満足感」「言葉を覚えるきっかけ（日英読み上げ）」 |
| プラットフォーム | モバイル/PCブラウザ。**特にiPad Safari での快適なプレイを最優先** |
| 公開方法 | GitHub Pages（静的ホスティング） |
| サーバー | **なし（完全サーバーレス）**。ログイン不要。データはブラウザ内に保存 |

### 設計の優先順位
1. **演出・アニメーションの気持ちよさ**（子どもが何度もタップしたくなること）
2. 動作の軽快さ（タップしてからの待ち時間が短い／カクつかない）
3. コレクションの満足感
4. 日英読み上げによる学びの要素

## 2. 技術要件

### 2.1 技術スタック（推奨）
- **TypeScript + Vite**（ビルド・開発サーバー）
- **アプリ全体の画面・レイアウト**（画面遷移／図鑑グリッド／ボタン／モーダル）は**素のDOM + CSS（flexbox/grid）**を基本とする。
- **タマゴ本体＋ひび割れ**は**SVG**でパスを描く（自然な卵のカーブ・ジグザグなひび線を表現しやすいため）。動き（揺れ・スクワッシュ&ストレッチ・パカッと開く）はSVG内のグループに対する**CSS `transform`**で付け、GPU合成の恩恵を受けつつ滑らかに動かす。SVGの`<filter>`や`d`属性そのもののアニメーションはiOS Safariで重くなりやすいため避ける。
  - キラキラ等のパーティクルは **CSS or Canvas** のどちらでも可。個数が多くなる場合（目安30〜40超）はCanvasを優先する。
- ガチャの抽選ロジック等の純粋関数は **Vitest** でユニットテスト可能にする。
- フレームワーク（React等）は必須ではない。導入する場合も**バンドルを軽く**保つこと。

### 2.2 アセット方針（重要）
- **モノ（動物・乗り物・食べ物など）の絵は、初期版では絵文字（Emoji）を採用する。**
  - 理由：画像ファイル・ライセンス・読み込み待ちが不要で、サーバーレス／即動作と相性が良い。色も鮮やかで子ども向け。全カテゴリ（🐘🚗🍎…）を1文字でカバーできる。
  - データ構造は後から画像（SVG/PNG）に差し替え可能な形にしておく（`emoji` フィールドを `assetUrl` に置換できる設計）。
- **効果音**は、初期版では **Web Audio API による簡単な合成音**（ポン、パカッ、キラーン等）で代替してよい。音声ファイルを使う場合は軽量なものに限る。
- タマゴ本体・UIは画像ファイルに頼らず描画する（具体的な役割分担は2.1参照：タマゴ＋ひびはSVG、それ以外はCSS）。
- **メイン（ガチャ）画面の背景**は、世界観づくりのため**ローカルにバンドルした風景画像**（`src/assets/scene_picture.jpg`／もりの中・恐竜のシルエット）を使用する。元PNG（約1.3MB）は高品質JPEG（mozjpeg q85・約124KB）へ圧縮して読み込みを軽くする。画像は Vite のビルドに含めて公開（サブパス・オフライン動作を維持）。文字やタマゴの視認性のため、画像の上に淡いスクリム（CSSグラデ）を重ねる。

### 2.3 データ永続化
- コレクション状況は **localStorage** に保存（サーバー不要）。
- 保存内容：所持アイテムID・各アイテムの取得個数・初回取得日時・設定（音声ON/OFF等）。
- 破損・未保存時は安全に初期状態へフォールバックすること。

### 2.4 GitHub Pages 対応
- Vite の `base` をリポジトリ名に合わせて設定（プロジェクトページの場合 `/<repo>/`）。
- ビルド成果物を Pages へ公開するワークフロー（GitHub Actions 推奨）も用意する。
- すべて相対パス／`import.meta.env.BASE_URL` を考慮し、サブパス公開でリンク切れしないこと。

## 3. 開発フェーズ（この順で進める）

> 各フェーズ終了時に「ブラウザで遊べる」状態にすること。

### フェーズ1：MVP（メイン体験）
タマゴをタップ → ガタガタ揺れる → ひびが入る → パカッと割れる → 中からモノが絵文字で登場 → キラキラエフェクト → 名前表示 → タップで次のタマゴへ。
- アイテムプールは仮で十数種類（各カテゴリ少しずつ）。
- 図鑑・レア・音声はまだ無くてよい。
- **この時点で GitHub Pages にデプロイして iPad で触れる**ことをゴールにする。

### フェーズ2：レアリティ
- レア度（後述）と重み付き抽選を実装。
- レアが出るときの**専用演出**（色・光・音・間の取り方が変わる）。

### フェーズ3：図鑑・コレクション
- 取得したモノを図鑑に登録。未取得はシルエット／「？」表示。
- localStorage 保存。コンプリート率・カテゴリ別フィルタ。

### フェーズ4：日英読み上げ
- 図鑑でモノをタップ → 日本語名 → 英語名の順に読み上げ（Web Speech API）。

### フェーズ5：仕上げ
- 効果音、NEWバッジ、設定画面、PWA対応（ホーム画面追加）、アクセシビリティ、演出の磨き込み。

## 4. 機能要件

### 4.1 メイン（ガチャ）画面 ★最重要
- 画面中央に**大きなタマゴ**を配置。待機中はわずかに揺れる／呼吸するような微アニメで「押したくなる」見た目に。
- **タマゴをタップ**すると、以下のシーケンスが再生される：

| 段階 | 演出 | 目安時間 |
|------|------|----------|
| ① 予兆 | タマゴがガタッと一瞬反応（スクワッシュ&ストレッチ） | 0.1〜0.2s |
| ② 振動 | だんだん激しくガタガタ揺れる | 0.8〜1.2s |
| ③ ひび割れ | 表面にひびが少しずつ広がる | ②に重ねる |
| ④ パカッ | 上半分が勢いよく開く＋一瞬のフラッシュ | 0.2s |
| ⑤ 登場 | 中からモノがポンッと跳ねて出る（バウンド） | 0.3〜0.5s |
| ⑥ キラキラ | モノの周囲にキラキラ粒子が弾ける | ⑤に重ねる |
| ⑦ 結果 | モノの名前を表示。「ゲット！」演出。図鑑に登録 | — |
| ⑧ 次へ | 画面タップで待機状態のタマゴに戻る | — |

- **連打しても破綻しない**こと（演出再生中の多重タップを無視 or スキップ可能に）。
- 子どもが飽きないよう、テンポは軽快に。出てくるモノは抽選で毎回変わる。

### 4.2 レアリティシステム
- レア度を**3段階**で定義（例）：
  - `common`（ノーマル）
  - `rare`（レア）
  - `superRare`（激レア）
- **重み付きランダム抽選**。例：common 80% / rare 18% / superRare 2%（数値は調整可能な定数として一箇所にまとめる）。
- 抽選は**シード可能な擬似乱数**で実装し、ロジックを Vitest でテスト可能にする（確率分布・境界値）。
- レア演出：
  - `rare` … 光の色変化・少し長めの溜め・専用の効果音・粒子増量。
  - `superRare` … 画面全体が虹色／金色に染まる、スローモーション気味の溜め、特別なジングル、最大級のキラキラ。
- レアが出た瞬間に「**やった！レアだ！**」と分かる強い手応えを与える。

### 4.3 図鑑（コレクション）
- 取得済みのモノを**グリッド**で一覧表示。
- **未取得**はシルエットまたは「？」で表示（集めたくなる動機付け）。
- 各モノに**日本語名・英語名・カテゴリ・レア度・取得個数**を持たせる。
- **カテゴリ別フィルタ**（動物／乗り物／食べ物 …）。
- 画面上部に**コンプリート率**（例：12 / 50 種類）。全コンプ時はお祝い演出。
- モノをタップ → 詳細（拡大表示・名前・レア度・個数・読み上げボタン）。

### 4.4 読み上げ（日英）
- 図鑑のモノをタップすると、**日本語名 → 英語名**の順に音声で読み上げる。
  - 例：🍎 をタップ → 「りんご」→「apple」
- ガチャでモノが**登場した瞬間**にも、同じく**日本語名 → 英語名**を読み上げる（読み上げON時）。
  - タマゴのタップ起点で読み上げを解錠（iOS対策）し、演出後の発話が弾かれないようにする。
- **Web Speech API（SpeechSynthesis）**を使用（サーバー不要）。
  - 日本語：`lang = "ja-JP"`、英語：`lang = "en-US"`。各言語の音声を `getVoices()` から選択。
  - **音声リストは非同期で読み込まれる**ため `voiceschanged` を待つ／リトライする実装にする。
  - **iOSでは最初の発話がユーザー操作（タップ）起点である必要**があるため、タップ駆動で問題なし。
  - 音声が利用できない端末では、名前テキスト表示にフォールバック。
- 設定で読み上げのON/OFF、言語の順序を切り替えられるとよい（任意）。

## 5. データ設計

### 5.1 アイテム定義（マスタデータ）
```ts
type Category =
  | "animal" | "sea" | "bug" | "vehicle"
  | "food" | "fruit" | "sweets" | "nature"; // 必要に応じて追加可
type Rarity = "common" | "rare" | "superRare";

interface ItemDef {
  id: string;        // 一意のID（例: "apple"）
  nameJa: string;    // 日本語名（例: "りんご"）
  nameEn: string;    // 英語名（例: "apple"）
  category: Category;
  rarity: Rarity;
  emoji: string;     // 表示（例: "🍎"）※将来 assetUrl に差し替え可能に
}
```
- データは **約100種類**を用意（どうぶつ／うみ／むし／のりもの／たべもの／くだもの／おかし／しぜん の8カテゴリにわたり、レア度も分散）。
- データは1つのファイル（例：`src/data/items.ts`）に集約し、追加・編集しやすくする。

### 5.2 セーブデータ（localStorage）
```ts
interface SaveData {
  collected: Record<string, {
    count: number;        // 取得個数（ダブり含む）
    firstSeenAt: number;  // 初回取得日時（NEW判定用）
  }>;
  settings: {
    speechEnabled: boolean;
    soundEnabled: boolean;
    reducedMotion: boolean; // 演出控えめモード
  };
}
```
- コンプリート率・NEW判定・ダブり数はこの保存データから導出する。

## 7. 追加機能の提案（採否は要相談 / 優先度付き）

### ぜひ入れたい（低コスト・効果大）
- **絵文字アセット採用**（2.2参照）：アセット調達不要で即リッチに見える。
- **NEW! バッジ**：初めて取得したモノに「NEW!」を表示。図鑑にも反映。
- **ダブり（重複）カウント**：同じモノを引いたら「×3」のように個数を表示。「2個目！」等の小演出。
- **未取得シルエット**：図鑑で集める動機付け。
- **コンプリート率・完成お祝い**：カテゴリ別／全体の達成感。
- **演出控えめモード（reduced-motion）**：激しい揺れが苦手な子・端末向け。OS設定 `prefers-reduced-motion` も尊重。

### 余裕があれば
- **PWA対応**：iPadのホーム画面に追加してフルスクリーンのアプリ風に。オフラインでも遊べる。
- **効果音**：タップ／パカッ／キラーン／レアジングル。Web Audioの合成音でMVP可。
- **天井（救済）演出**：一定回数レアが出ないと次回レア確定。子どもが飽きない配慮（任意）。
- **設定でガチャ確率・テンポを調整できる隠し設定**（保護者・開発用）。
- **背景の季節・時間バリエーション**などの彩り。

### 非推奨 / 注意
- **触覚フィードバック（バイブ）**：iOS Safari は `navigator.vibrate` 非対応。実装しても無効になるため、手応えは**音・光・動き**で表現する。
- 課金・広告・ログイン・外部通信は**入れない**（サーバーレス／子ども向けの方針）。

## 8. 非機能要件・iPad/Safari 固有の注意点

- **パフォーマンス**：アニメーションは `transform` / `opacity`（GPU合成）中心で。レイアウト再計算を避ける。パーティクル数は端末負荷を見て調整。
- **タッチ操作**：hover前提のUIにしない。タップ領域は**大きく**（子どもの指でも押しやすく）。
- **ビューポート**：`meta viewport` に `viewport-fit=cover`。ダブルタップズーム／ピンチで演出が乱れないよう調整。
- **セーフエリア**：iPadのノッチ／角丸を考慮（`env(safe-area-inset-*)`）。
- **音声・オーディオの制約**：iOSはユーザー操作起点でのみ再生可能。Web Audio の `AudioContext` は最初のタップで `resume()` する。
- **PWA（任意）**：`manifest.json` ＋ `apple-mobile-web-app-capable` 等でホーム画面追加・スタンドアロン表示に対応。
- **アクセシビリティ**：文字は大きく・コントラスト確保。`prefers-reduced-motion` 尊重。読み上げが無理な環境ではテキスト表示でフォロー。
- **オフライン**：初回読み込み後はネット接続なしで遊べることが望ましい（PWA採用時）。

## 9. テスト方針

- **ユニットテスト（Vitest）対象**：
  - レアリティ抽選（確率分布・シード固定での再現性・境界値）。
  - セーブ/ロード（破損データのフォールバック、NEW判定、ダブり集計、コンプリート率の導出）。
  - アイテムマスタの整合性（IDの一意性、必須フィールドの欠落チェック）。
- アニメーション・演出・読み上げは目視（実機）確認とする。
- 抽選は**シード可能な擬似乱数**にして、テストの決定性を確保すること。

## 10. 完成イメージ（受け入れ基準の例）

- iPadのSafariで開き、大きなタマゴをタップすると、ガタガタ→パカッ→モノ登場→キラキラ、までが**気持ちよく**再生される。
- たまにレアが出て、明らかに演出が豪華になる。
- 図鑑を開くと集めたモノが並び、未取得は「？」。タップで日本語→英語の音声が鳴る。
- アプリを閉じて開き直しても、集めたモノが残っている。
- GitHub Pages のURLを共有すれば、誰でもログイン不要で遊べる。

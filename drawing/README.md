# 🎨 こども おえかき

iPad で遊べる、子供向けのお絵描き Web アプリです。
**バックエンド不要・外部 API 不要**の静的サイトなので、GitHub Pages にそのまま置いて公開できます。オフラインでも動きます。

## ✨ できること

| 機能 | 説明 |
| --- | --- |
| ✏️ ペン | 指やペンで自由に描く |
| 🌈 にじいろ | 描くと色が虹色に変わる |
| 🧽 けしごむ | 描いた絵を消す |
| 🪣 ぬりつぶし | 囲まれた所をバケツ塗り |
| ⭐ スタンプ | 絵文字スタンプ／ステッカー（🎲 ランダムもあり） |
| 🎨 ぬりえ | 線画テンプレートを呼び出して色ぬり |
| 🖼️ はいけい | 無地・方眼・水玉に加え、イラスト背景（こうえん/ゆうえんち/うみ/うちゅう/まち/ぼくじょう/ゆき/おばけ）。背景の上に絵を描いて遊べる |
| ↩️ ↪️ | もどる／やりなおし（Undo / Redo） |
| 🗑️ ぜんけし | 全消し（確認あり・誤操作防止） |
| 💾 ほぞん | PNG で保存 |
| 🔊 おと | 効果音オン／オフ |
| ⛶ ぜんがめん | フルスクリーン |

線の太さは 4 段階、色は 12 色。ボタンは大きく、子供が誤操作しにくい UI です。

**iPad の横画面**を主対象にしています。描くスペースを最大化するため、
「いろ」「ふとさ」などはボタンを押すとツールバーの上にポップアップで出る方式にしました
（パネル表示中にキャンバスをタップすると、描かずにパネルが閉じます）。縦画面でも崩れずに使えます。

## 🚀 つかいかた（ローカル）

ES モジュールと Service Worker を使うため、`file://` で直接開くと動きません。簡単なローカルサーバーで開いてください。

```bash
# どれか1つ（このフォルダ内で実行）
python -m http.server 8000
# または
npx serve .
```

ブラウザで `http://localhost:8000` を開きます。

## 🌐 GitHub Pages へ公開

1. このフォルダの中身をリポジトリにそのまま push する
2. GitHub の **Settings → Pages** で、Branch を `main`（フォルダ `/root`）に設定
3. 数十秒後に `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます

> パスはすべて相対指定（`./...`）なので、サブディレクトリ公開でもそのまま動きます。

iPad の Safari では、共有メニューから **「ホーム画面に追加」** するとアプリのように全画面で遊べます（PWA / オフライン対応）。

## 📁 ファイル構成

```
drawing/
├── index.html          画面の骨組み
├── manifest.json       PWA 設定
├── service-worker.js   オフライン用キャッシュ
├── icon.svg            アプリアイコン
├── css/
│   └── style.css       見た目（大きいボタン・カラフル）
└── js/
    ├── app.js          メイン制御（全体をつなぐ）
    ├── data.js         色・スタンプ・ぬりえ等のデータ定義
    ├── scenes.js       イラスト背景（公園・遊園地など）をSVGで定義
    ├── canvas.js       3レイヤー管理（背景／絵／線画）
    ├── history.js      もどる・やりなおし
    ├── fill.js         バケツ塗り（フラッドフィル）
    └── sound.js        効果音（Web Audio で合成）
```

キャンバスは **3 レイヤー構成**です。

- `bg-layer` … 背景
- `draw-layer` … ユーザーが描く絵（Undo / 塗り の対象）
- `overlay-layer` … ぬりえの線画（常に一番上・触れない）

## 🔧 機能を増やすには

- **色を足す** … `js/data.js` の `COLORS` に追記
- **スタンプを足す** … `js/data.js` の `STAMPS` に絵文字を追記
- **ぬりえを足す** … `js/data.js` の `TEMPLATES` に SVG（線だけ）を追記
- **無地・模様の背景を足す** … `js/data.js` の `BACKGROUNDS` に追記
- **イラスト背景を足す** … `js/scenes.js` の `SCENES` に `{ id, label, type:'scene', base, svg }` を追記
  （`cloud()` `tree()` `sun()` などの共通パーツを組み合わせると簡単）

データを書き換えるだけで UI に反映されます。ファイルを更新したら、`service-worker.js` の `CACHE = 'oekaki-v1'` のバージョン名を変えると、確実に新しい内容が配信されます。

## 🛡️ iPadで誤操作を防ぐ設定（おすすめ）

お絵描き中に、画面の端からのスワイプや複数指の操作で中断しないようにする設定です。
アプリ側でブラウザのピンチ拡大・ダブルタップ拡大・ページのスクロールは止めていますが、
**iOS本体のスワイプ（下から・上からのコントロールセンター／Dock／アプリ切替）は
Webページからは止められません**。次の iOS 設定で完全に防げます。

### 1. ホーム画面に追加して全画面で使う
Safari でアプリを開く → 共有ボタン → **「ホーム画面に追加」**。
追加したアイコンから開くと、Safari のバー無し・全画面で動き、左右端スワイプの戻る操作も無効になります。

### 2. アクセスガイド（Guided Access）でロックする ← いちばん効果的
1. **設定 → アクセシビリティ → アクセスガイド** をオンにする。
2. 「パスコード設定」で専用パスコードを決める（指紋/Face IDも可）。
3. お絵描きアプリを開く → **ホームボタンを3回押す** → 「開始」。
4. これで、下から・上からのスワイプ、コントロールセンター、Dock、アプリ切替などが
   すべて無効になり、子供がアプリから出られなくなります。
5. 解除は **ホームボタンを3回押す**→パスコード入力 →「終了」。

> 左下の「オプション」で、画面の特定エリアのタッチ無効化や、モーション無視なども設定できます。

### 3. （任意）マルチタスクのジェスチャをオフ
**設定 → 一般 → マルチタスクとジェスチャ** で「4本指・5本指でのジェスチャ」をオフにすると、
複数指でのアプリ切替を防げます。

## 📝 メモ

- iPad Safari の保存は、端末・バージョンによって「写真に追加」ではなく新規タブで画像が開く場合があります。その場合は画像を長押しして保存してください。
- 効果音は外部ファイルではなく Web Audio で合成しているため、オフラインでも鳴ります。

## ステップ 8：iPhone（狭い画面）対応（実装済み）
ご指摘「iPhoneだと積み場所が狭すぎてまともに遊べない」への対応。原因は縦持ちで
上部の注文バー（見本タワーが縦に伸びる）とパレット（折り返し）に高さを奪われ、
真ん中の積み場所(`.stage`)が潰れていたこと＋アイスが固定84pxで収まらないこと。
- [x] ①積み場所に合わせてアイスを自動リサイズ（`js/main.js` の `fitBuilder()`）。
      `.stage` の実寸と最大段数(`MAX_SCOOPS`)からサイズを逆算し CSS変数 `--scoop` に設定。
      初回＋`resize`／`orientationchange`（iOS向けに遅延再測定）で更新。
      `css/style.css`：`.scoop`／`.stack .scoop`／`.cone` を `--scoop` ベースに変更。
      横画面用 `@media (max-height:560px)` のアイス固定サイズは撤去（JSが調整するため競合回避）。
- [x] ②縦画面の上部バーを低く保つ（`@media (orientation: portrait)`）。
      見本（`.sample .scoop.small`／`.sample-cone`）を小さくして、注文段数が増えても
      上部が積み場所を圧迫しないように（下→上の積み方の見え方は維持）。
- [x] ③縦持ちスマホでパレットをコンパクト化（`@media (orientation: portrait) and (max-width:600px)`）。
      `.palette-scoop` 44px・ラベル小・余白詰めで折り返しを減らす（指のタップ目標は確保）。
- [x] ドラッグ中ゴーストの中心合わせをサイズ追従に（`js/input.js`）。`--scoop` 可変に合わせ
      生成後に実寸から `ghostHalf` を算出（固定42pxをやめた）。
- [ ] 実機（iPhone Safari 縦／横）で、最大段数まで積めること・追従性・回転時の崩れを確認 ← **要確認**

## ステップ 9：iPhone 17 など縦長スマホ向けレイアウト最適化（実装済み）
ご要望「タブレット最適化の現状を、スマホ（iPhone 17）最適化のパターンにしたい」。
方針はレスポンシブ強化（iPad横画面＝主対象は壊さず、スマホで開くと自動で最適化）。
すべて `css/style.css` のみで対応（HTML/JS は変更なし。`fitBuilder` がアイスサイズを
自動調整するので段数フィットは従来どおり機能する）。
- [x] セーフエリア対応（Dynamic Island／ノッチ・下部ホームインジケータを避ける）。
      `viewport-fit=cover`（既設）+ `env(safe-area-inset-*)` を `max()` で最低余白と併用。
      対象：`.sound-btn`（上・右）／`.sidebar`（上・左）／`.palette`（下・左右）／
      `.mode-select`（四辺）。env 非対応の iPad/PC では従来値のままで無影響。
- [x] 縦持ちスマホ（≤480px）専用にヘッダーを2段化（`@media (orientation: portrait) and (max-width:480px)`）。
      1段目＝お客さん＋見本（吹き出し）、2段目＝スコア・ヒント・操作ボタンを中央寄せ横一列。
      狭幅での横詰まり／ボタンの折り返しの乱れを解消。タップ目標は約44pxを維持。
      ※タブレット縦（幅600〜768）は従来どおり1行レイアウトのまま。
- [x] 縦持ちスマホのパレット圧縮（`max-width:600px`）に下部セーフエリア余白を追加し、
      ホームインジケータと味ボタンが重ならないように。
- [ ] 実機（iPhone 17 / Safari 縦・横）でセーフエリア余白・2段ヘッダー・回転時の崩れを確認 ← **要確認**
